Editor-first flow

The agent lives where the code is.

01

Select code

Use normal Helix selection. DoomHelix attaches the selected block, file path, language, cursor, diagnostics, workspace root, and visible editor state.

02

Ask naturally

`:agent explain`, `:agent fix`, `:agent refactor`, `:agent edit`, or `:agent chat` route through one active ACP session.

03

Approve changes

Tool calls and file edits go through editor prompts, so agent power stays visible and controlled.

Built for serious sessions

Not a wrapper. A real editor integration.

DoomHelix avoids the throwaway `codex exec` loop. It talks ACP, keeps a live session, streams into a selectable transcript, and understands which file or selection you mean.

Context on every turn

Active file, selections, language, diagnostics, workspace root, command history, and theme metadata.

Movable panel

`:agent position left`, `right`, `top`, or `bottom`; resize with `:agent resize +5` and `-5`.

ACP backends

Use Codex ACP, Claude Code ACP, both, or configure a custom ACP-compatible agent.

Patch escape hatch

Direct edits are the main path, but diff preview and `git apply` workflows are still available.

Flat config

No nested server maze.

The common path is one active ACP agent. `command` implies stdio; `url` implies WebSocket. 

[editor.agent]
enable = true
name = "claude"
command = "claude-code-acp"
args = []
panel-position = "right"
panel-size = 30

Install

One command, then choose your agent.

The installer asks for Codex, Claude, both, or custom ACP later. It keeps existing config files untouched and installs npm-based adapters under `~/.local` by default. 

curl -fsSL https://raw.githubusercontent.com/borngraced/doom-helix/agentic-acp-integration/install.sh | sh
