Agentic Helix, native ACP

DoomHelix

A focused Helix fork for coding with Codex, Claude, or any ACP-compatible agent. It keeps editor context close, asks before risky tool use, and gives the agent one clean panel that can move with you.

01

Context that means something

DoomHelix sends the active file, real selections, cursor, diagnostics, workspace root, visible buffer state, theme, and recent command context.

02

One agent panel

One transcript, one route for messages, no duplicate scratch chaos. Move it left, right, top, or bottom and resize it during the session.

03

Real ACP sessions

Codex ACP and Claude Code ACP run as long-lived backends with streaming messages, permission requests, and direct editor-aware workflows.

Flat config

One active agent. No server maze.

`command` means stdio. `url` means WebSocket. The default install uses amberwood. 

theme = "amberwood"

[editor.agent]
enable = true
name = "claude"
command = "claude-code-acp"
args = []
panel-position = "right"
panel-size = 30

Install

Pick Codex, Claude, both, or custom ACP later.

The installer writes `dhx`, installs the selected ACP adapter when needed, keeps npm installs under `~/.local`, and leaves existing config files untouched. 

curl -fsSL https://raw.githubusercontent.com/borngraced/doom-helix/agentic-acp-integration/install.sh | sh
